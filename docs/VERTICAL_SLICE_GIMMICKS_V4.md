# HorroLive Vertical Slice & Gimmick Direction v4

## 0. 目的

この文書は、企画ボードの3つのキービジュアルを「プレイヤーが操作して理解できる遭遇」に変換する。

- A: **右上にだけ映っている**
- B: **バズるほど近づいてくる**
- C: **コメント欄が先に気づく**

基準は [Creative Direction v4](CREATIVE_DIRECTION_V4.md)。現行実装の6怪異を増やすのではなく、3本の代表遭遇へ制作密度を集中する。

## 1. 現状評価

### 1.1 すでに使える基盤

- MainとPIPは同一`SceneSnapshot`を使う。
- PIPは400–700ms遅延、12–18fpsで表示できる。
- フリーズ、反復、スキップ、露出、横ずれ、フォーカスぼけがある。
- `DORMANT → TELEGRAPH → ACTIVE → resolution → AFTERMATH → COMPLETE` の状態機械がある。
- 撮影判定は遅延PIP上の対象、距離、中央フレーム、ライト、電池を参照する。
- Chatの追従停止、新着件数、スクロール保持がある。
- 視聴者数の4段階と一度だけの閾値発火がある。
- Pause、Journal、Chat drawer、ページ非表示でシミュレーションを止められる。

### 1.2 核を弱めている問題

| 問題 | 現状 | v4判断 |
| --- | --- | --- |
| 怪異が同型 | 6体がほぼ「近づく→9秒以内に撮る」 | 3遭遇で異なる観測と選択を作る |
| 予兆が短い | 多くが520ms | PIP遅延より長い1.5–2.5秒 |
| Chatがランダム | 2–4秒ごとの汎用文 | 怪異単位の決定論的台本 |
| 接近が数値だけ | Tierでノイズと台詞が増える | 同一観測者の空間距離を変える |
| Tier0不可視発火 | 見えず撮れない怪異が進行し得る | 表示可能TierまでDORMANT |
| 後退が無視にならない | 前方通過か別scene離脱のみ | 進行方向に依存しない離脱判定 |
| 電池表示が不一致 | HUD可変、PIPはCAM 88%固定 | 共通RIG BATTERYへ統一 |
| 成功が白フラッシュ | 祝祭的で怖さが切れる | 露出落ち、無音、timecode固定 |
| 章ごとに閾値再通知 | 高Tierでまとめて再発火し得る | vertical sliceはstream単位ledger |
| 型が未使用 | `channelTimeline`、`chatScript`等に実データなし | 3遭遇を完全データ化 |

## 2. Slice構成

目標尺は5分30秒、許容4–7分。時間はプレイヤーの停止と再確認で伸縮する。

| 時間 | 場所 | Main | PIP | Chat | プレイヤーの学習 |
| --- | --- | --- | --- | --- | --- |
| 0:00–0:25 | 暗部調整／配信開始 | ライト境界を確認 | 同じ壁を遅れて表示 | SYSTEMのみ | 三画面の配置 |
| 0:25–0:55 | 病院入口 | 生活痕跡を照らす | 湿った床の反射が0.6秒遅れる | 通常会話 | 止まると差を比較できる |
| 0:55–1:25 | 校正ポイント | 粉塵が流れる | 一粒だけ静止 | 「今止まった？」 | CAPTUREと誤撮影 |
| 1:25–2:35 | 遭遇A | ロッカー奥は空 | 右上に観測者 | 位置の曖昧な警告 | カメラだけの真実 |
| 2:35–3:05 | 患者室前 | 観測者は消える | 遅延フレームだけ背後に残る | 一時的に通常へ戻る | 撮影が危険を上げる |
| 3:05–4:15 | 遭遇C | 掲示は一見普通 | 文字の一部だけ欠落 | 沈黙→同文→ハンドル | Chatも疑う |
| 4:15–5:25 | 遭遇B | 出口と暗い廊下 | 観測者が肩越しへ接近 | 人間投稿が止まる | 配信継続か退出か |
| 5:25–6:00 | 終了／結果 | 外へ出るか映像を向ける | 最終フレーム | SYSTEMの終了時刻 | 記録と生存の代償 |

校正ポイントは主要怪異に数えない。初回CAPTURE成功時に237から2,370へ上げ、遭遇AをPIPで観測可能にする。

## 3. 共通遭遇契約

### 3.1 状態

```text
DORMANT
  → TELEGRAPH 1.5–2.5s
  → ACTIVE 7–12s
  → RECORDED | IGNORED | MISSED | RETREATED
  → AFTERMATH 1.0–2.0s
  → COMPLETE
```

- `RETREATED` は「怪異から後退して安全距離へ出る」選択。`IGNORED` と区別する。
- `RECORDED` だけが正解ではない。無視と離脱でも章は進む。
- `MISSED` は即死ではなく、証拠欠落、Chat信頼度、観測者距離のいずれかを変える。
- 主要遭遇の同時`ACTIVE`は1件。
- Pause中はworld、anomaly、viewer、Chat、音響台本の時計をすべて止める。

### 3.2 予兆

意味のあるペナルティには、異なるチャンネルで最低2つの手掛かりを置く。

例:

1. Chatが一度止まる。
2. PIPの露出だけが下がる。
3. Mainの環境音から蛍光灯音が消える。
4. 遮蔽物の縁だけが濡れて光る。

ノイズの増加だけを予兆にしない。

### 3.3 撮影結果

| 結果 | 即時 | 後続 |
| --- | --- | --- |
| 正しい撮影 | 250ms無音、timecode固定、ARCHIVE追加 | 視聴Tier上昇、観測者が近づく |
| 誤撮影 | RIG -4%、AF迷い、懐疑コメント | 次のcapture windowを0.5秒遅らせる |
| 無視 | 対象を通過、証拠なし | 視聴Tier維持、Chatの一部が不満 |
| 後退 | 安全距離へ戻る | その遭遇だけ再試行可、リスク上昇なし |
| 見逃し | window終了 | 後続ヒントの確度低下、即死なし |

白い全面フラッシュ、拍手音、ポイント加算の祝祭表示を使わない。

## 4. CAL-01「遅延の校正」

### 狙い

怪異を出す前に「PIPは同じ世界の少し前」を教える。失敗しても進行できる非主要イベント。

### 台本

| 相対時刻 | Main | PIP | Chat／Audio |
| --- | --- | --- | --- |
| 0.0s | ライト内の粉塵が右へ流れる | 通常遅延 | 足音のみ |
| 0.8s | 粉塵が通過 | 一粒だけ画面中央で静止 | 「いま一個止まった？」 |
| 1.8s | 変化なし | 静止粒子に弱いAF枠 | SYSTEM: DVR sync unstable |
| 3.0–8.0s | CAPTURE可能 | 中央へ合わせられる | 追加ヒント1件 |

### 結果

- 成功: 237 → 2,370。`CAL_DVR_GHOST`をARCHIVEへ追加。
- 失敗／無視: SYSTEMが校正を完了し、2,370へ上げる。ただし証拠は追加しない。

視聴Tierを上げるためにプレイヤーを詰ませない。

## 5. 遭遇A「右上の空所」

### ID

`event_pip_corner_observer`

### 狙い

企画ボードAをそのまま操作へ変換する。Mainには最後まで観測者を描かず、PIPの右上だけで成立させる。

### トリガー

- scene: `hospital-ward-a`
- risk: Tier 1以上
- playerがロッカーの260–360px手前
- 同時`ACTIVE`なし

Tier 0では完全に`DORMANT`。見えない状態で緊張、音、罰を発火しない。

### タイムライン

| 相対時刻 | Main | PIP | Chat | Audio |
| --- | --- | --- | --- | --- |
| 0.0s | ロッカー奥は空 | 通常 | ambient停止 | 蛍光灯音を切る |
| 0.8s | 変化なし | 露出が一度沈む | 「今、止まった？」 | イヤーピースに短い接触音 |
| 1.6s | 変化なし | 右上にalpha 0.22の輪郭 | 「右上」「棚の奥？」 | 無音 |
| 2.4s | ライトがロッカーを照らす | 輪郭はライトと無関係に残る | 3件の短いburst | AFが輪郭のない顔を探す |
| 3.0–11.0s | CAPTURE可能 | alpha 0.30まで上げる | 誤誘導を1件混ぜる | 低い布擦れ |

### 解決

- `RECORDED`: 観測者の遠景証拠。Risk Tier 2へ。
- `IGNORED`: 遅延PIPの最後の一枚だけ、現在の主人公背後に輪郭を残す。Tier維持。
- `RETREATED`: 一度だけ15秒後に再試行。
- `MISSED`: Chatの人間／怪異投稿の識別が一段難しくなる。

### 合格条件

- Mainへの観測者描画呼び出しが0回。
- PIP遅延700msでも2秒以上観測可能。
- reduced effectsでもalpha差とChat位置情報が残る。

## 6. 遭遇C「コメントが名前を読む」

### ID

`event_chat_reads_first`

### 狙い

Chatを攻略字幕ではなく、第三の観測チャンネルにする。実個人情報は使わず、固定ゲーム内`PLAYER_HANDLE`だけを扱う。

### トリガー

- 患者室入口
- 遭遇Aが`COMPLETE`
- 掲示板`ward_notice_board`を画面内へ入れる

### Chat状態

```text
AMBIENT 2–4s
→ SILENCE 2.0s
→ EVENT_BURST 4件 / 3.0s
→ HIJACK 5.0s
→ RECOVER
```

### タイムライン

| 相対時刻 | Main | PIP | Chat | Audio |
| --- | --- | --- | --- | --- |
| 0.0s | 古い掲示、読めない染み | 同じ掲示 | 投稿停止 | 通知音の残響だけ |
| 2.0s | 変化なし | 1文字分の空白 | 3人が別の位置を指す | 部屋鳴りを狭める |
| 3.2s | ライトで`PLAYER_HANDLE`が一瞬見える | 文字は欠落したまま | 同文が時刻違いで2回 | ピンマイク接触 |
| 5.0s | 文字が消える | AF枠が掲示へ出る | SYSTEM風投稿「名前を消せ」 | 通知音を逆再生 |
| 6.0–14.0s | CAPTURE可能 | 掲示差を保持 | Human 1件＋Corrupted 2件 | 低い呼吸 |

### 解決

- `RECORDED`: ARCHIVEにMain/PIP差分を対で保存。Risk Tier 3へ上げる選択を可能にする。
- `IGNORED`: 視聴Tier維持。次の遭遇のHuman hintを1件失う。
- `RETREATED`: Chatは通常へ戻るが、掲示の染みだけ残る。
- `MISSED`: 次のSYSTEM時刻が23:47で固定される。実時間や端末時刻を使わない。

### 合格条件

- イベント台本がambient乱数より必ず優先される。
- `jammed`中はHuman投稿を止める。怪異投稿だけを許可する場合は状態で区別する。
- 実ユーザー名、IP、地域、端末、アカウント情報への参照がない。

## 7. 遭遇B「同接距離」

### ID

`event_viewer_distance_climax`

### 狙い

視聴者数をスコア表示から物理的な脅威へ変える。観測者は通常のCAPTURE対象ではなく、プレイヤーが撮った証拠の結果として近づく。

### 距離契約

| Tier | PIP内の観測者 | Main bleed | Chat |
| --- | --- | --- | --- |
| 0 / 237 | 不在 | なし | ambient |
| 1 / 2,370 | 背景の奥、約420px相当 | なし | 位置の指摘 |
| 2 / 23,700 | 遮蔽物の後ろ、約180px | 120–180msの肩／裾 | 同文反復 |
| 3 / 237,000 | 主人公の肩越し、60–90px | 200–300msの部分像 | Human停止、HIJACK |

Risk ledgerはvertical slice全体で一つ。章変更で閾値通知を再発火しない。

### クライマックス

1. 出口`exit_door`がMainに入る。
2. PIPは0.6秒前の廊下を映し、観測者が主人公より出口側に立つ。
3. ChatのHuman投稿が止まる。
4. プレイヤーは次のどちらかを選ぶ。

| 選択 | 入力 | 結果 |
| --- | --- | --- |
| EXIT | 出口へ移動し`E` | 配信終了。生存優先、証拠は未完成 |
| KEEP STREAMING | PIPで観測者を中央へ入れ`CAPTURE` | 237,000へ到達。最終フレームが肩越しで固定 |

選択UIをモーダルで出さず、移動と視線で決めさせる。

### 合格条件

- 視聴Tier変更と観測者距離が一度だけ同期する。
- どちらの選択も即時ゲームオーバーにしない。
- 237,000到達を紙吹雪、明るい赤、ポイント音で祝わない。
- reduced effectsでは揺れを減らし、距離、Chat停止、顔認識枠を残す。

## 8. PIP効果の台本化

現行のランダム劣化はambientへ限定し、主要効果は遭遇データから発火する。

| 効果 | 通常上限 | 用途 | reduced effects |
| --- | --- | --- | --- |
| FREEZE | 250–900ms | 観測者の位置確定 | 250–400ms |
| REPEAT | 2–4frames | 過去が残る感覚 | 2frames |
| SKIP | 1–3frames | 接近の欠落 | 1frame |
| EXPOSURE | ±12% | 輪郭を一度だけ浮かす | ±6% |
| DISPLACE | 2–8px | 侵食 | 2px |
| AF_HUNT | 0.6–1.2s | 空白の顔を探す | 枠移動のみ |
| FALSE_FACE_BOX | 1–2個 | 人のいない位置を顔判定 | 枠を低速表示 |

強度を重ねず、一つの手掛かりに一つの主要効果を割り当てる。

## 9. 優先度

### P0 / vertical slice成立に必須

1. 3遭遇の`AnomalyDefinition`実データ
2. event Chat scheduler
3. Tierから観測者距離へのstream単位遷移
4. TELEGRAPH 1.5–2.5秒への延長
5. Tier0不可視発火の停止
6. 方向に依存しないRETREATED／IGNORE判定
7. RIG BATTERY表示統一
8. 成功白フラッシュの置換
9. flashlight occlusion
10. snapshot遅延とpauseの統合テスト

### P1 / P0後

- 車椅子の向きが見ていない間だけ変わる
- 光限定型の1–3frame怪異
- 校正オーブの演出強化
- 患者室への出入り
- 暗部キャリブレーション、Help、Settings

### P2 / 本編拡張

- 最奥礼拝室／慰霊棚
- 追加ロケーション4キット
- 複数エンディング
- 人形、逆さ男、巨大影を別怪異として戻す判断
- 長編収集物と自由入力Chat

## 10. テスト

### 自動

- PIP専用観測者がMain render listへ入らない。
- Tier 0で`event_pip_corner_observer`が`DORMANT`を維持する。
- 400ms、700msの両遅延でACTIVE windowが2秒以上残る。
- 同一scene内の後退で`RETREATED`へ入る。
- Chatの`SILENCE`中にHuman投稿が増えない。
- EVENT_BURSTがambientをpreemptする。
- 各viewer thresholdがstream中一度だけ発火する。
- Pause中はworld、anomaly、Chat、riskの時刻が進まない。
- PIPとHUDのbattery値が一致する。

### 視覚

- 1440×900
- 1366×768
- 1024×768
- 844×390
- 667×375
- 390×844
- 375×667

すべてでPIPが見え、Mainの手掛かり、Chatのイベント、CAPTURE操作が互いに隠れないことを確認する。

### プレイテスト質問

1. 最初に「何か変だ」と思ったのはMain、PIP、Chatのどれか。
2. 自分から停止して比較した瞬間はあったか。
3. 撮影が危険を上げると、どの演出で理解したか。
4. Chatを信じられないと思った具体的な投稿は何か。
5. 最後にEXITとCAPTUREのどちらを選び、なぜか。

正解率だけでなく、比較行動が自発的に起きたかを成功指標にする。
