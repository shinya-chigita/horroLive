# 『LIVE』AI組織型開発ワークフロー

更新基準: `main@11929e7` 以降  
現在工程: **改善プロトタイプ**  
次工程: Vertical Slice（品質ゲート2と人間オーナー承認の後のみ）

## 1. 管理原則

この開発は日数や消化率ではなく、前提条件、依存関係、成果物、品質チェック、品質ゲート、人間承認、修正履歴で管理する。前工程の必須成果物が基準を満たさない限り、次工程へ進めない。

判断優先順位は次の通り。

1. 最後まで遊べる
2. 操作が快適
3. ライト内が見える
4. ゴールが分かる
5. カメラを見る意味がある
6. コメント欄がゲームと連動する
7. 怖い
8. 見た目が良い
9. 要素が多い

## 2. 確定仕様

- 2D横スクロール、1人用、ジャンプなし、戦闘なし
- 左右移動、ライト照準、調べる、PIP確認、撮影、必要時の走行・隠密
- Mainでは見えない異変が、同じ空間を映すPIPには現れる
- コメントがプレイヤーより先に異変へ反応する
- 撮影成功だけが同接を増やし、同接増加が観測者の接近と危険度を上げる
- ライト中心は読め、扉・棚・柱・カーテンが光と撮影判定を遮る
- 改善プロトタイプは「侵入→異変→撮影→配信異常→逃走→脱出→LIVE残留」を3〜5分で伝える
- コメントは事前生成・タグ・イベント駆動とし、実行時生成AIへ依存しない

確定仕様の変更、スコープ変更、品質ゲート通過、リリース可否は人間オーナーだけが承認する。

## 3. 現工程のスコープ

品質ゲート2の正式な検証対象は、盤面選択にある **「改善プロトタイプ / 病院STANDARD」** の専用経路だけとする。

専用経路は通常の病院全編とは別の4区間・4怪異・収集物なしの実行グラフを使用する。通常病院、廃校、周回モードの挙動変更は、改善プロトタイプ修正へ混ぜない。

以下の既存要素は削除せず凍結し、ゲート2通過まで追加制作・調整の対象にしない。

- 廃校盤面
- Deep Broadcast
- 周回解放と永続進行
- 複数エンディングの拡張
- 新しい章、怪異、収集物、分岐

## 4. 実装の正本

- 実行入口: `src/main.tsx` → `src/AppV2.tsx`
- 探索と入力: `src/components/MainGameView.tsx`
- PIP: `src/components/PipCameraV2.tsx`
- 盤面定義: `src/game/boardDefinitions.ts`
- コメントイベント: `src/game/broadcastEventDirector.ts`
- ゲート2専用規則: `src/game/improvementPrototype.ts`

`src/App.tsx` など旧コンポーネントは履歴互換用であり、新規修正の対象にしない。別AIは正本を確認してから作業を始める。

## 5. AI組織の責任

| 組織 | 主責任 | 必須成果物 |
| --- | --- | --- |
| 統括ディレクター | コア体験、仕様間整合、過剰設計の監査 | 統合レビュー、矛盾一覧、ゲート判定案 |
| プロジェクト管理 | 依存関係、状態、承認、変更履歴 | タスク表、未決事項、リスク、ゲート進捗 |
| ゲームデザイン | ライト→PIP→コメント→撮影→同接→危険のループ | ループ、状態遷移、パラメータ、イベント連動 |
| レベルデザイン | 3〜5分の一本道と目的の段階化 | フロー、配置、トリガー、チェックポイント |
| シナリオ・コメント | ヒント、ノイズ、侵食の状況連動 | コメントデータ、条件、終端演出 |
| アート/UI/サウンド | 視認性、操作、情報優先度、予兆 | 表現仕様、UX項目、音響表 |
| エンジニアリング | 実装可能な状態・データ・イベント制御 | コード、技術仕様、ビルド、既知不具合 |
| QA | 機能、進行、視認性、操作、連動、体験 | 再現手順、重要度、実結果、ゲート資料 |

## 6. タスク状態と修正ループ

状態は `未着手 / 前提待ち / 作成中 / AIレビュー中 / 人間確認待ち / 修正中 / 完了 / 保留 / 不採用` を使用する。

1. 未完了タスクと前提を抽出する
2. 担当AIが成果物を作る
3. 統括ディレクターが企画整合を確認する
4. エンジニアリングが実装する
5. QAが検査し、S/A/B/Cへ分類する
6. 不合格項目を同工程へ戻す
7. 自動チェックと実機試遊を再実施する
8. 品質ゲート資料を更新する
9. 人間オーナーが承認または差し戻す

新しい案は実装へ混ぜず、提案内容、理由、影響範囲、コスト、現工程での必要性、後回し可否を別記する。
