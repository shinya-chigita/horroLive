## 1. North Star

> 見えていないのは、こっちかもしれない。

恐怖は怪異そのものではなく、同じ時刻・同じ場所を観測しているはずの三つのチャンネルが一致しないことから生まれる。

1. **Main / 肉眼**: 現在の世界。懐中電灯の外はほぼ読めない。
2. **PIP / 配信カメラ**: 同じ世界を 400–700ms 遅れて映し、肉眼にない層を含む。
3. **Chat / コメント欄**: 人間の集団観測。先に気づく、誤認する、沈黙する、乗っ取られる。

プレイヤーの基本行動は次の一文に収める。

> 歩く → 違和感を受け取る → 立ち止まって三画面を比較する → 撮る・無視する・離れる → 注目と危険が変わる

ただ暗い横スクロール、ただ怪異を撮るゲーム、ただコメントが流れる配信画面のどれか一つになった時点で本作らしさを失う。

## 2. 体験の約束

### 2.1 プレイヤーに感じてほしい順序

1. 「何か変だが、どこが変か分からない」
2. 「自分よりコメント欄の方が先に気づいた」
3. 「PIPは過去を映しているのに、こちらを見返している」
4. 「撮れ高を得るほど、それが近づいてくる」
5. 「配信を続けているのは自分か、配信そのものか」

### 2.2 ホラー強度

- 予兆なしの即死を使わない。
- 意味のあるペナルティの前に 1.5 秒以上の観測可能な予兆を置く。
- 全身・正面・均一照明の怪異を同時に成立させない。
- フルスクリーン白フラッシュを成功報酬に使わない。
- 大音量、急拡大、強い画面揺れを同時に重ねない。
- 流血、欠損、内臓、武器、戦闘を恐怖の主役にしない。

### 2.3 対象と尺

- vertical slice 全体: **4–7分**
- 一つの主要遭遇: **60–90秒**
- P0 の主要遭遇: 3本
- 一度に `ACTIVE` になる主要怪異: 原則1体

旧資料の「60–90秒で全機能を見せる」は採用しない。比較して考える時間を確保するため、60–90秒は一遭遇の検証尺とする。

## 3. 世界と物語の姿勢

### 3.1 廃病院 vertical slice

舞台は「廊下2–3モジュール＋患者室1室」に絞る。最奥祭壇、5章、6怪異、複数エンディングは縦切りの核が成立した後に戻す。

病院の怖さはオカルト記号の多さではなく、生活と労働の痕跡が停止していることから作る。

- 半分飲まれた紙コップ
- 書きかけの患者記録
- 片方だけ残ったスリッパ
- 充電ケーブルの刺さった壊れた端末
- 押し込まれた薬品カート
- 消毒液の輪染み
- 何度も開閉されたロッカーの擦れ

各画面には最低一つの「生活の痕跡」と一つの「説明できない差」を置く。

### 3.2 後続ロケーション

P1 以降の4キットは廃旅館、廃校、古い宗教施設、地下通路。各キットは背景差し替えではなく、観測差のルールを一つずつ持つ。

| 場所 | 生活の痕跡 | 固有の観測差 |
| --- | --- | --- |
| 廃旅館 | 宿帳、湯呑み、布団、内線電話 | PIPだけ襖の枚数が違う |
| 廃校 | 日直欄、上履き、教材、放送機器 | Chatだけ読める黒板の名前 |
| 古い宗教施設 | 掃除道具、供物の容器、参拝記録 | 光の角度で像の向きが変わる |
| 地下通路 | 工具、保守札、排水跡、非常灯 | 音と映像の進行方向が逆になる |

## 4. 主人公

### 4.1 役割

主人公は戦う人ではなく、撮るために少しだけ無理をする配信者である。勇敢な英雄にも、泣き続ける被害者にも寄せない。平静を装う癖が崩れていく過程を姿勢で見せる。

### 4.2 造形

- 暗い短髪。輪郭は不揃いだが、巨大なアニメ髪にしない。
- 頭部は全高の32%以下。
- 短いフィールドジャケット、細身のパンツ、重い靴。
- 胸部カメラ、斜めストラップ、バックパック、腰ポーチをシルエットで識別可能にする。
- 右手の電池式懐中電灯が最も明るい小物。
- イヤーピース／ピンマイクはChatと外界をつなぐ記号。
- モバイルバッテリーは配信リグ全体の共通電源として装備に含める。
- 武器、軍装、過剰なサバイバル装備を足さない。

### 4.3 感情段階

| 段階 | 姿勢 | 顔 | 呼吸／動作 |
| --- | --- | --- | --- |
| 余裕 | 胸が開く | 口角をわずかに上げる | 片手でライトを振れる |
| ビビる | 肩が上がる | 目線だけ先行 | 一歩だけ後ろへ重心 |
| 焦り | 前傾 | 瞬きが減る | ライトの追従が早まる |
| 疲弊 | 首と肘が落ちる | まぶたが重い | 歩幅が狭くなる |
| 恐怖 | 身体を縮める | 顔をPIPへ向け切れない | ライトを両手で固定 |

上下のバウンスは1–2論理pxまで。歩行8fps、走行10fps。マスコット的な跳ねを禁止する。

## 5. 主怪異「観測者」

### 5.1 ひとつの存在を近づける

怪異を数で増やすより、一体の「観測者」がチャンネルをまたいで接近する方を優先する。制作IDは `anom_observer`。

- 性別・年齢を断定できない。
- 病衣と作業用エプロンの境界に見える輪郭。
- 顔は描き込まず、カメラの誤顔認識枠だけが顔の存在を主張する。
- 肩、指先、衣服の裾のいずれか一つだけに動きを持たせる。
- 画面へ焼き付けるノイズや走査線は怪異画像へ含めず、PIP後処理で加える。

### 5.2 リスク段階

| Tier / 視聴者 | Main | PIP | Chat |
| --- | --- | --- | --- |
| 0 / 237 | 不在 | 背景の圧縮崩れだけ | 一人だけ違和感を述べる |
| 1 / 2,370 | 不在 | 遠景、alpha 0.22–0.30 | 「右上」など位置だけ共有 |
| 2 / 23,700 | 120–300msの部分的bleed | 遮蔽物の後ろ、主人公の背後 | 同文反復、沈黙、誤誘導 |
| 3 / 237,000 | 全身は出さない | 肩越し、顔認識枠、画面侵食 | 人間の投稿が止まり、固定ハンドルを呼ぶ |

視聴者数は成功スコアではない。増加は祝いではなく、通知音が濁る、赤表示が一瞬遅れる、観測者との距離が縮むことで伝える。

### 5.3 補助怪異

- **光限定型**: Mainでライトが横切る1–3フレームだけ見える。PIPには残らない。
- **遮蔽差型**: 40–70%を隠し、MainとPIPで向きか位置だけが違う。
- **環境差型**: 車椅子、掲示、ベッドなど既存物の状態差。新しい全身怪異より優先する。

## 6. 光と闇

### 6.1 基準

- Main論理解像度: **640×300**
- 床基準: `y = 238`
- 暗部: 画面の70–80%を相対輝度0.03未満
- 強いハイライト: 画面の約2%以下
- 赤: 通常1%未満
- ライト色: `#D1B68B`
- ライト全角: 50–64°
- 到達距離: 220–300論理px
- 照準追従: 90–140msの慣性
- 環境光: 一画面一か所、alpha 0.06–0.12

### 6.2 遮光

壁、ロッカー、カーテン、ベッド、扉枠は光を遮る。遮蔽物の後ろを単純な黒塗りにせず、エッジに1–2pxの湿った反射か埃を残し、奥行きを読めるようにする。

照明レイヤー順:

1. 背景面
2. 世界小物
3. 怪異チャンネル
4. 主人公
5. 前景遮蔽物
6. 光マスク
7. チャンネル別後処理

| 最深部 | `env.void` | `#03060A` |
| 基本背景 | `env.black` | `#0A0A0A` |
| 冷たい壁 | `env.wall` | `#1F2529` |
| 灰緑構造 | `env.structure` | `#2A2F2B` |
| 境界線 | `ui.border` | `#242824` |
| 主文字 | `ui.text` | `#D4D3D4` |
| 補助文字 | `ui.muted` | `#6E665B` |
| ライト | `light.warm` | `#D1B68B` |
| 錆・染み | `material.rust` | `#6E5132` |
| LIVE／危険 | `signal.red` | `#8A2026` |
| カメラ異常青 | `noise.blue` | `#30424C` |
| カメラ異常紫 | `noise.violet` | `#121822` |

明るい純白、鮮やかなネオン赤、青い魔法光を常用しない。色でしか状態を伝えないUIを禁止する。

## 8. カメラ映像

- PIP表示: 320×180
- 同一canonical sceneから取得し、別の背景を描かない。
- 遅延: 400–700ms
- 表示更新: 12–18fps
- ニアレストネイバー拡大
- 通常ノイズは弱く、手掛かりの輪郭を壊さない。
- 怪異固有の台本でフリーズ、反復、スキップ、露出、横ずれ、AF迷い、誤顔認識枠を発火する。
- ランダムノイズだけで怖がらせない。
- `CAM xx%` はヘッダーの電池値と必ず一致させる。

怪異の見せ方は「暗いから見えない」ではなく、「見えているはずなのにチャンネルが一致しない」を優先する。

## 9. UIと用語

### 9.1 情報序列

1. Main
2. PIP
3. Chat
4. LIVE／視聴者／章／電池の一行ステータス
5. 必要時だけ開くARCHIVE、HELP、SETTINGS

### 9.2 固定用語

| 意味 | 表示 |
| --- | --- |
| 配信継続状態 | `LIVE / REC` |
| 瞬間の撮影行為 | `CAPTURE / 撮影` |
| 収集物と証拠 | `ARCHIVE / 記録帳` |
| 配信リグ共通電源 | `RIG BATTERY / BAT` |

「記録」を撮影ボタンと記録帳の両方に使わない。

### 9.3 採否

- Chatは縦切りでは読み取り専用。入力欄は採用しない。
- 好評価数は常設しない。必要ならイベント中のChat本文として扱う。
- 配信時間はLIVE状態と並べて残す。
- Health数値は常設しない。
- Tension数値は出さず、呼吸・姿勢・音・PIP不安定で示す。
- 角丸、ガラスカード、光沢CTA、派手なグラデーションを使わない。

## 10. 電源ルール

電池はカメラとライトで別管理せず、装備のモバイルバッテリーにつながった**配信リグ共通電源**とする。

- 開始値: 100%
- ライト点灯: 約0.12%/秒を基準に調整
- 正しい撮影: 固定消費なし、または1%以内
- 誤撮影: -4%
- 0%: ライト消灯、撮影不能。PIPは最後の低照度フレームを短時間保持して停止
- 回復: vertical sliceでは一度だけ、環境内バッテリーで+25%

4–7分の通常プレイで、乱発しなければ20–40%を残す。数字を難易度だけでなく「撮るか温存するか」の判断へ接続する。

## 11. Chatの文体

Chatは字幕ではない。情報の確度を投稿主体ごとに変える。

| 種別 | 文体 | 役割 |
| --- | --- | --- |
| Human | 短い、口語、互いに少し食い違う | 集団観測 |
| Hint | 位置や変化を述べるが答えを断定しない | 停止と比較を促す |
| System | 無感情、時刻・回線・人数 | 現実の基準 |
| Corrupted | 句読点、時刻、同文が壊れる | 侵食 |

イベント中は `AMBIENT → SILENCE → EVENT_BURST → HIJACK → RECOVER` を明示的に遷移させる。露骨な壁文字「右上ヲ見ロ」で答えを教えず、複数コメントの不一致から気づかせる。

固定主人公ハンドルを使う場合は `PLAYER_HANDLE` とし、実ユーザー名、IP、地域、端末、アカウント情報を取得・表示しない。

## 12. 音

- 常時音楽を敷かず、空調、蛍光灯、靴音、布、呼吸、通知の質感で空間を作る。
- 三画面の不一致の前に1.5–3秒の静寂を置く。
- 正しい撮影は成功音より、タイムコード固定と短い無音で示す。
- 観測者の接近音を足音に限定しない。衣擦れ、マイク接触、イヤーピースの逆位相を使う。
- Master、環境音、効果音、ミュートを用意する。
- 意味のある音には画面上の代替手掛かりを置く。

## 13. 禁止事項

- グロ、過剰な血、欠損を主な見世物にする
- 予兆なしの即死、連続ジャンプスケア
- 怪異を正面・全身・明所で長時間見せる
- 武器、戦闘、追跡AIを作品の中心にする
- 明るい色を広い面積に使う
- 情報量の多いHUD、常時表示の数値ゲージ
- MainとPIPへ無関係なランダム背景を出す
- ノイズを素材画像へ焼き込み、手掛かりまで潰す
- 実在ユーザーの個人情報を使う侵食演出
- 既存ホラー記号を理由なく足す「祭壇」「血文字」「白い少女」の量産

## 14. 成立判定

プレイテストで以下を満たせば、方向性が成立している。

- 初見プレイヤーが主要遭遇ごとに一度以上、自発的に停止してPIPかChatを見る。
- PIP専用怪異をMainだけで発見できない。
- 手掛かりを見失っても、再観測または無視の選択ができる。
- 正解撮影、誤撮影、無視、後退に異なる結果がある。
- 視聴者増加が報酬だけでなく距離の危険として理解される。
- Chatが単なる攻略字幕ではなく、疑う対象になる。
- reduced effectsでも同じ手掛かりと解決経路が残る。
- 画面の暗さが操作不能ではなく、意図した不確実性として機能する。
