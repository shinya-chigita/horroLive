# 品質ゲート2 — 改善プロトタイプ

検証対象: **病院STANDARD / 品質ゲート用経路**  
開始方法: 盤面選択 → `品質ゲート用ビルドを開始`  
基準時間: PLAYING開始から最終脱出まで180〜300秒  
現在判定: **HOLD / 実ブラウザ完走済み・人間オーナー承認と残課題確認待ち**

## 1. ベースライン監査

`main@11929e7` は自動チェックに合格していたが、次の理由で品質ゲート2を差し戻した。

- `chase` 分岐が到達不能で、短い逃走が存在しない
- 撮影を一度も使わずクリアできる
- 最初の怪異を見逃しても同接が増え、撮影との因果が崩れる
- MISSED / IGNOREDにも「記録済み」と誤案内する
- 通常脱出ではLIVE残留が発生せず、残留時も全UIが表示される
- 2盤面、周回、複数エンディングが混在し、3〜5分の検証経路が分離されていない

## 4. 自動品質結果

- `npm run lint`: 合格
- `npm test`: 131 / 131 合格
- `npm run build`: 合格
- 新規純粋ロジック: 4区間の隔離経路、必須撮影ゲート、解決状態別の失敗文、180〜300秒判定、到達可能なCHASE分岐、必須マイルストーン順序、同接TENSION下限、怪異演出のTENSION反映

PLAYING中かつ非停止状態だけを有効時間として加算する。Journal、Chat、タブ非表示、GAMEOVER、回転ダイアログの停止時間は除外する。PLAYING開始、各必須撮影、CHASE開始・成功、脱出、終了の経過秒と順序判定は、最終LIVE要素の `data-prototype-*` 属性へ記録する。改善プロトタイプのLIVE画面は自動終了しない。

React統合、Canvas上の視認性、実所要時間、初見理解は単体試験では判定しない。

2026-07-15の実ブラウザ通しでは `234203ms / IN_TARGET / sequence-complete=true` を実測した。ただし、追跡解除後の安全地点で130秒の観察を追加しており、熟練操作の自然な到達時間は約2分だった。初見プレイとして無理なく180〜300秒になるかは人間確認を残す。

## 5. 人間オーナー試遊チェック

1. 盤面選択から「品質ゲート用ビルドを開始」する
2. 最初の10秒で移動、ライト照準、撮影を理解できるか
3. ライト中心の床・扉・棚・怪異を即認識できるか
4. MainとPIPが同じ場所を映していると感じるか
5. コメントを見てPIPを自発的に確認するか
6. 未撮影時は同接237のままで、撮影後だけ2,370以上へ増えるか
7. 足跡、PIP限定人影、反復する車椅子の3撮影を通らない限り先へ進めず、再挑戦できるか
8. 同接上昇で観測者が近づき、終盤のCHASEへ自然につながるか
9. 逃走が短く明確で、理不尽な死亡にならないか
10. 脱出後にLIVE以外が消え、タイトル回収として伝わるか
11. 最終LIVE要素の `data-prototype-duration-band` が `IN_TARGET`（180〜300秒）か
12. PC、ゲームパッド相当入力、スマホ横画面でUIが重ならないか

全項目を満たした場合のみ、人間オーナーが `品質ゲート2：承認` を記録する。不合格時はVertical Sliceへ進まず、この経路だけを修正する。

## 6. Ultracode監査（2026-07-15）

最新 `main@e196e2e84743d3e25af3b14b6bf6efa1b29d7d74` を起点に、設計・実装・QAの3監査を並列実施した。詳細は [品質ゲート2 Ultracode監査報告](./QUALITY_GATE_02_ULTRACODE_AUDIT_2026-07-15.md) を参照する。

監査で修正した主な問題は、改善プロトタイプ最初の足跡がベッド遮蔽と失効距離の組み合わせで撮影不能だった進行停止、低FPS時の有効時間過少計測、フォーカス外のグローバル入力、タブ非表示復帰時のRAF時間飛び、PIP操作後のフォーカス喪失、スマホ横の目的UI重なりと再表示タイマー、現在バッテリーとPIP表示の不一致である。

最終判定は **HOLD**。S/A級の既知不具合は修正済みだが、自然な3〜5分尺、reduced-motion実機エミュレーション、スマホでの目的再表示→閉じる操作、実タブhidden→resumeの通し確認を人間ゲート前提として残す。このため自動マージ条件を満たさない。
